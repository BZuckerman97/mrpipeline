# Clump instruments using LD reference

Computes pseudo p-values (to ensure correct ranking by PLINK) and
performs LD clumping via
[`ieugwasr::ld_clump()`](https://mrcieu.github.io/ieugwasr/reference/ld_clump.html).

## Usage

``` r
clump_instruments(
  dat,
  rsq_thresh,
  clump_kb = 10000,
  bfile = NULL,
  plink_bin = NULL,
  pop = "EUR"
)
```

## Arguments

- dat:

  Data frame with columns `rsid`, `pval`, `id`, and optionally `beta`
  and `se` (used for pseudo p-value computation). If `beta` and `se` are
  absent, `pval` is used directly.

- rsq_thresh:

  R-squared clumping threshold.

- clump_kb:

  Clumping window in kb. Default 10000.

- bfile:

  Path to PLINK bfile prefix. If `NULL`, uses API clumping.

- plink_bin:

  Path to PLINK binary. If `NULL`, auto-detected.

- pop:

  Population for API clumping. Default `"EUR"`.

## Value

Data frame of clumped variants (output of
[`ieugwasr::ld_clump()`](https://mrcieu.github.io/ieugwasr/reference/ld_clump.html)).

## Details

Pseudo p-values are generated by:

1.  Computing z-scores: `beta / se`

2.  Computing precise -log10(p) via
    [`Rmpfr::pnorm()`](https://rdrr.io/pkg/Rmpfr/man/distr-etc.html)

3.  Ranking SNPs by -log10(p) descending

4.  Assigning evenly spaced pseudo p-values from 1e-100 to 0.9
